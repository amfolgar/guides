Semaphore

Project
  1. Create a new project in Semaphore

  2. Login to your Semaphore account (if you haven't been added to OK GROW!'s account, ask to be added.)
  3. Select your project (listed as okgrow / [project-name])
  4. Edit project Settings by clicking "Project settings":
    1. Build Settings
      1. Set the "Node.js version" to "node.js 7.10.1" (or later)
      2. Add these lines under setup: Semaphore build settings
    2. Environment Variables
      1. Add these variables (you'll get the values for these from the "Database Information" document created above):
        • PRODUCTION_DB_HOST
        • PRODUCTION_DB_PORT
        • PRODUCTION_DB_NAME
        • PRODUCTION_DB_USERNAME
        • PRODUCTION_DB_PASSWORD
        • STAGING_DB_HOST
        • STAGING_DB_PORT
        • STAGING_DB_NAME
        • STAGING_DB_USERNAME
        • STAGING_DB_PASSWORD
        • AWS_ACCESS_KEY_ID (optional)
        • AWS_SECRET_ACCESS_KEY (optional)
        • APPNAME (optional - used on AWS setup)
    3. Configuration Files
      1. Add the application's settings.json file
Staging Server

On the Semaphore project page click “Set Up Deployment”.

  1. Select "Heroku":
    1. Select "Automatic"
    2. Select the "master" branch
  2. Enter the API key for the accounts+semaphoredeploy@okgrow.com Heroku user (from here)
  3. Select the Staging Heroku app for this project from the list; name the server “Staging”
  4. On the Semaphore project page, under "Servers", click the server name (e.g., "Staging")
  5. On the servers screen, click the "Edit Server" button
  6. Under "Deploy commands" click the "Change deploy commands" link and paste the contents from staging app deploy config
Production Server

On the Semaphore project page click the + button beside "Servers" to add a new server.

  1. Select "Heroku":
    1. Select "Automatic"
    2. Select the "master" branch
  2. Enter the API key for the accounts+semaphoredeploy@okgrow.com Heroku user (from here)
  3. Select the Production Heroku app for this project from the list; name the server “Production”
  4. On the Semaphore project page, under "Servers", click the server name (e.g., "Production")
  5. On the servers screen, click the "Edit Server" button
  6. Under "Deploy commands" click the "Change deploy commands" link and paste the contents from production app deploy config.

Slack Integration

  1. Go to the project's Slack channel and, under "Channel Settings" click "Add an app or integration"
  2. Add Sempahore as an integration and copy the web hook URL; save the integration
  3. On Semaphore, go to Project Settings / Notifications / Webhooks

  4. Add the web hook URL and select "Build and Deploy" from the "Receive After" dropdown

  5. Currently you can follow the steps taken for the Rapunzl project.

MongoDB Atlas

If the project is using Galaxy to host the app the MongoDB Atlas is the recommended hosting provider. If you already have an account it will take you about 5 mins to setup. You can find detailed step-by-step instructions to getting setup and started in our blog post here.

Atlas is the best option if wishing to scale CPU & Memory independently to it's storage & disk I/O, cheapest for using the WiredTiger Storage engine, encrypting data at rest, have 3 to 7 data bearing nodes, continuous backups & snapshots.

Semaphore config:

  1. Create new project
    1. NOTE: Paul needs to do this step currently. Will fix...
  2. Edit Build Settings:
    1. Go to https://semaphoreci.com/okgrow/<PROJECT_NAME>/settings
    2. Set the Node version to node.js 7.10.1 (or later)
    3. Add these lines under setup: Semaphore build settings
    4. Note: You may also need to add more project specific steps.
  3. Edit Project Settings:
    1. Under "Configuration Files" add the project's settings.json file.
    2. Under "Configuration Files" add the project's app-production.json file.
    3. Under "Configuration Files" add the project's app-staging.json file.
    4. Under "Branches" change the cancellation strategy to "queued and started builds".
  4. On the Sempahore project homepage click “Set Up Deployment” (staging)
    1. Choose Heroku, then automatic, and then master.
    2. Enter the API key for the accounts+semaphoredeploy@okgrow.com Heroku user.
    3. Select the Staging Heroku app for this project from the list, and name it “Staging”.
    4. Add the staging app deploy config
  5. Add a new server by clicking the + button beside 'Servers' (production)
    1. Choose Heroku, then manual, and then master.
    2. Enter the API key for the accounts+semaphoredeploy@okgrow.com Heroku user, check our Accounts.
    3. Select the Production Heroku app for this project from the list, and name it “Production”
    4. Add the production app deploy config
  6. Add Slack notifications (no email)
    1. Go to the project's slack channel and add an integration
    2. Add sempahore as an integration and make note of the webhook URL, then save the integration
    3. Go to Project Settings / Notifications / Webhooks in Semaphore
    4. Add the webhook URl and select 'Build and Deploy' from the 'Receive After' dropdown

Heroku config

  1. In Heroku: Under ‘Access’, add accounts+semaphoredeploy@okgrow.com as a collaborator in the staging and production Heroku app
